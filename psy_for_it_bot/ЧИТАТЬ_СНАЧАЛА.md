# С чего начать (для того, кто получил этот комплект)

## 1. Открой этот файл первым

В **корне** выгруженной папки он называется **`ЧИТАТЬ_СНАЧАЛА.md`** — это тот же текст, что и `docs/START_HERE_RECIPIENT.md` в репозитории.

---

## Если у тебя Windows (типично для приёмки)

1. Установи и **запусти Docker Desktop** (иконка кита в трее) — пошагово в **`ИНСТРУКЦИЯ.md`**, шаг 1. Без запущенного Docker команды ниже не сработают.
2. Положи папку в удобное место, например `C:\Projects\psy_for_it_bot\` (или клонируй монорепо `portfolio_projects` и открой подпапку `psy_for_it_bot`). В проводнике открой эту папку, кликни в **адресную строку**, напиши `cmd` и нажми Enter — откроется командная строка **уже в нужной папке**.
3. Команды **`docker compose ...`** вводи **так же**, как в этом файле (в современном Docker Desktop это работает из CMD и из PowerShell). Альтернатива для первого запуска — двойной клик по **`setup.bat`** в корне папки (описано в `ИНСТРУКЦИЯ.md`).
4. Разделы про **`usermod`**, **`sudo`** и **permission denied** к сокету Docker относятся к **Linux / WSL**. На Windows с Docker Desktop это обычно **не нужно**.

Скрипт **`export_handover_directory.sh`** запускает только тот, кто **собирает** комплект (разработчик, чаще Linux / WSL / macOS). Ты получаешь **готовую папку** или архив и дальше следуешь этой инструкции.

## 3. Перед запуском

1. **Windows:** **Docker Desktop** — см. `ИНСТРУКЦИЯ.md`, шаг 1. **Linux / WSL:** Docker Engine + Compose — там же.
2. Только **Linux / WSL:** если Docker пишет **permission denied** к сокету — группа `docker` или `sudo`, см. «Проблемы и решения» в `ИНСТРУКЦИЯ.md`.
3. Скопируй **`.env.example`** → **`.env`**, открой в редакторе и заполни как минимум:
   - **`BOT_TOKEN`** — от @BotFather
   - **`OWNER_TELEGRAM_IDS`** — твой числовой Telegram ID (например через @userinfobot)
4. **`DATABASE_URL`** и **`REDIS_URL`** для запуска в Docker обычно **не меняй** — они уже согласованы с `docker-compose.yml`. При необходимости смотри комментарии в `.env.example`.

**Не коммить и не отправляй публично** файл `.env` с настоящим токеном.

---

## 4. Сборка и запуск бота

В терминале зайди в **корень** папки с ботом и выполни:

```bash
docker compose up --build -d
```

Посмотреть последние логи бота:

```bash
docker compose logs --tail=120 bot
```

На Windows в инструкции также описан вариант с **`setup.bat`**.

---

## 5. Автотесты (обязательно перед приёмкой)

Нужны **оба** файла: `docker-compose.yml` и `docker-compose.dev.yml`.

```bash
docker compose -f docker-compose.yml -f docker-compose.dev.yml build tests
docker compose -f docker-compose.yml -f docker-compose.dev.yml run --rm tests
```

Ожидается примерно **23 passed** (если число в проекте изменилось — смотри вывод и `SESSION_STATE.md` в комплекте).

---

## 6. Если что-то пошло не так

Кратко:

- **Порт 5432 занят** — на Windows тоже бывает (другой Postgres или старый контейнер); останови лишнее, см. `ИНСТРУКЦИЯ.md`.
- **Permission denied у Docker** — в основном Linux / WSL; на Windows проверь, что **Docker Desktop запущен**.
- **Фиксированное имя проекта** `mental-support-bot` в Compose — при копии папки тома БД могут совпасть со старой установкой; детали в `ИНСТРУКЦИЯ.md`.

Удачной проверки.
