# 📖 Mental Support Bot — Инструкция по установке

> Эта инструкция написана простым языком — для людей, которые не занимаются программированием.
> Следуй шагам по порядку, и всё получится.

---

## Что это за бот?

**Mental Support Bot** — закрытый Telegram-бот для корпоративных клиентов.
Он позволяет сотрудникам компании анонимно обращаться к mental health консультантам прямо в Telegram.

**Возможности:**
- 🚨 Неотложная помощь — связь с консультантом в течение 90 минут
- 📅 Запись на плановую сессию с выбором времени
- 👥 Группы поддержки
- 📊 Аналитика для владельца (Owner)
- 🌐 Три языка: Русский, English, Українська
- 🔑 Доступ только по инвайт-кодам — закрытая система

## Шаг 1 — Установи Docker Desktop

Docker — это программа, которая запускает бота в изолированной среде. Она включает всё необходимое: базу данных, хранилище и самого бота.

1. Перейди на сайт: **https://www.docker.com/products/docker-desktop/**
2. Нажми **"Download for Windows"**
3. Запусти скачанный файл и установи (Next → Next → Install)
4. После установки **перезагрузи компьютер**
5. Запусти Docker Desktop — в трее (правый нижний угол) появится иконка кита 🐳

> ⚠️ Docker должен быть **запущен** (иконка в трее) каждый раз, когда ты хочешь запустить бота.

---

## Шаг 2 — Получи токен для Telegram бота

Токен — это уникальный ключ доступа к боту. Его нужно получить один раз.

1. Открой Telegram и найди бота: **@BotFather**
2. Напиши ему `/newbot`
3. Введи **имя бота** (например: `PSY for IT Support`)
4. Введи **username бота** (только латинские буквы, должен заканчиваться на `bot`, например: `psy_for_it_bot`)
5. BotFather пришлёт тебе **токен** — длинная строка вида:
   `1234567890:ABCdefGHIjklMNOpqrSTUvwxYZ`
6. **Скопируй этот токен** — он понадобится на следующем шаге

> 🔒 Токен — это пароль твоего бота. Никому не передавай его.

---

## Шаг 3 — Узнай свой Telegram ID

Telegram ID — это твой уникальный числовой идентификатор. Нужен чтобы назначить тебя администратором (Owner) бота.

1. Открой Telegram и найди бота: **@userinfobot**
2. Напиши ему `/start`
3. Он покажет строку: `Id: 123456789` — это и есть твой ID
4. **Запомни или скопируй** это число

## Шаг 5 — Настрой файл .env

Файл `.env` — это настройки бота. В нём хранятся токен, пароли от базы данных и т.д.

1. Перейди в папку с ботом
2. Найди файл `.env.example` и **скопируй** его
3. Переименуй копию в `.env` (без `.example`)
4. Открой `.env` в блокноте (правая кнопка → Открыть с помощью → Блокнот)
5. Заполни поля:

```
BOT_TOKEN=сюда_вставь_токен_от_BotFather
DATABASE_URL=postgresql+asyncpg://postgres:postgres@db:5432/mental_support
REDIS_URL=redis://redis:6379/0
OWNER_TELEGRAM_IDS=сюда_вставь_свой_telegram_id
```

**Пример заполненного файла:**
```
BOT_TOKEN=1234567890:ABCdefGHIjklMNOpqrSTUvwxYZ
DATABASE_URL=postgresql+asyncpg://postgres:postgres@db:5432/mental_support
REDIS_URL=redis://redis:6379/0
OWNER_TELEGRAM_IDS=987654321
```

> ⚠️ `DATABASE_URL` и `REDIS_URL` **не трогай** — они уже настроены правильно для Docker.
> Если у тебя несколько владельцев, перечисли их через запятую:
> `OWNER_TELEGRAM_IDS=111111111,222222222`

6. Сохрани файл (Ctrl+S)

## Шаг 6 — Запусти бота

### Быстрый способ (Windows):
1. Дважды кликни на файл **`setup.bat`** в папке с ботом
2. Скрипт автоматически проверит Docker и запустит бота
3. Дождись сообщения **"Бот успешно запущен!"**

### Ручной способ:
1. Открой папку с ботом
2. В адресной строке проводника напиши `cmd` и нажми Enter
3. Введи команду:
   ```
   docker-compose up --build -d
   ```
4. Первый запуск занимает 3–5 минут (скачивает компоненты)
5. После завершения — бот работает

## Шаг 7 — Проверь что бот работает

1. Открой Telegram
2. Найди своего бота по username (который ты указал в BotFather)
3. Напиши `/start`
4. Должно появиться меню выбора языка

> ✅ Если меню появилось — всё работает!
> ❌ Если бот не отвечает — смотри раздел "Проблемы и решения" ниже

## Управление ботом

### Просмотр логов (что делает бот):
Открой папку с ботом в CMD и введи:
```
docker-compose logs -f bot
```
Нажми Ctrl+C чтобы выйти из просмотра логов.

### Запуск автотестов (важно)
Тестовый сервис описан в `docker-compose.dev.yml`, поэтому нужны **два compose-файла**:
```
docker compose -f docker-compose.yml -f docker-compose.dev.yml build tests
docker compose -f docker-compose.yml -f docker-compose.dev.yml run --rm tests
```

Проверить список сервисов (если сомневаешься, что `tests` виден):
```
docker compose -f docker-compose.yml -f docker-compose.dev.yml config --services
```

### Остановить бота:
```
docker-compose down
```

### Запустить снова (после остановки):
```
docker-compose up -d
```

### Перезапустить (если что-то пошло не так):
```
docker-compose restart bot
```

### Обновить бота после изменений в коде:
```
docker-compose up --build -d
```

## Как пользоваться ботом

### Роли в системе

| Роль | Кто это | Как получить доступ |
|------|---------|-------------------|
| **Owner** | Администратор | Telegram ID в `.env` файле |
| **Consultant** | Психолог/консультант | Owner добавляет вручную |
| **User** | Сотрудник компании | По инвайт-коду |

### Первые шаги для Owner (администратора):

1. Напиши боту `/start` — автоматически станешь Owner
2. Зайди в меню **"Компании"** → создай компанию
3. Укажи лимит пользователей и дату контракта
4. Нажми **"Генерировать инвайт-коды"** — бот пришлёт файл с кодами
5. Раздай коды сотрудникам компании

### Добавление консультанта:

1. Узнай Telegram ID консультанта (через @userinfobot)
2. В меню Owner → **"Консультанты"** → **"Добавить"**
3. Введи Telegram ID и имя консультанта
4. Отправь консультанту ссылку на бота — при `/start` он автоматически получит роль консультанта

### Как сотрудник получает доступ:

1. Получает инвайт-код от HR/Owner
2. Переходит по deep-link: `https://t.me/ИМЯ_БОТА?start=КОД`
   или вводит код вручную в боте
3. Выбирает язык — и готово!

## Проблемы и решения

### ❌ Бот не отвечает
**Причины и решения:**
- Docker не запущен → запусти Docker Desktop (иконка кита в трее)
- BOT_TOKEN неверный → проверь токен в `.env`
- Бот упал → посмотри логи: `docker-compose logs bot`

### ❌ Linux / WSL: Docker пишет `permission denied` (к сокету Docker)
На Linux и в WSL пользователь должен иметь право обращаться к демону Docker.

**Вариант 1 (правильный для постоянной работы):** добавить себя в группу `docker`, затем перелогиниться или выполнить `newgrp docker` в новом терминале:
```
sudo usermod -aG docker "$USER"
```
После этого снова открой терминал и повтори `docker compose …`.

**Вариант 2 (быстро, но каждый раз с паролем):** запускать команды с `sudo`, например:
```
sudo docker compose up --build -d
```

### ❌ Порт 5432 уже занят (`address already in use` / Postgres не стартует)
Порт **5432** часто занят **другим PostgreSQL** на компьютере или **старым** стеком `docker compose` из другой папки.

**Что сделать:** останови лишний Postgres на хосте **или** останови старый проект: в его папке выполни `docker compose down` (или останови контейнер, который держит 5432). Затем снова запусти бота из папки этого проекта.

### Имя проекта Docker Compose и смена папки
В `docker-compose.yml` задано фиксированное имя проекта: **`mental-support-bot`**. Оно **не** меняется, если ты переименовал или скопировал папку на диске. Из-за этого Docker может подцепить **те же тома** (данные БД), что и у прежней копии. Если нужна «чистая» база в новой копии — об этом лучше заранее договориться с тем, кто настраивал Docker, или использовать `docker compose down -v` **осознанно** (это удалит данные в томах этого проекта).

### ❌ "docker-compose: команда не найдена"
Установи Docker Desktop (Шаг 1). Если установлен — перезагрузи компьютер.

### ❌ "no such service: tests"
Ты запустил тесты только с `docker-compose.yml`.
Используй команду с двумя файлами:
```
docker compose -f docker-compose.yml -f docker-compose.dev.yml run --rm tests
```

### Профили окружения: dev и prod
- Для локальной разработки используй `.env` (обычно `APP_ENV=dev`, rate-limit можно держать выключенным).
- Для продакшна используй шаблон `.env.prod.example` и включай:
  - `APP_ENV=prod`
  - `INVITE_RATE_LIMIT_ENABLED=true`

### ❌ "Error: database connection failed"
База данных не запустилась. Попробуй:
```
docker-compose down
docker-compose up --build -d
```

### ❌ Бот показывает ошибку "access_denied"
Твой Telegram ID не совпадает с тем, что в `.env`. Проверь `OWNER_TELEGRAM_IDS`.

### ❌ Хочу перенести бота на другой ПК
1. Скопируй всю папку с ботом
2. Скопируй файл `.env` (в нём твои настройки!)
3. Установи Docker на новом ПК (Шаг 1)
4. Запусти `setup.bat` или `docker-compose up --build -d`

> ⚠️ **Важно:** Данные базы данных (пользователи, сессии) хранятся в Docker-томах.
> При переносе на другой ПК данные **не переносятся** — придётся начать с нуля.
> Если нужно перенести данные — обратись к разработчику.

## Автозапуск бота при включении ПК

Чтобы бот запускался автоматически при старте Windows:

1. Создай файл `autostart.bat` в папке с ботом:
   ```bat
   @echo off
   cd /d "%~dp0"
   docker-compose up -d
   ```
2. Нажми Win+R, введи `shell:startup`
3. Скопируй ярлык `autostart.bat` в открывшуюся папку

> Docker Desktop должен быть настроен на автозапуск с Windows
> (Настройки Docker → General → "Start Docker Desktop when you log in")
