# Чеклист ручного прогона U/C/O (short_tz baseline)

## Подготовка
- Поднять проект: `sudo docker compose up --build -d`
- Проверить лог старта: `sudo docker compose logs --tail=120 bot`
- Для dev в `.env`: `INVITE_RATE_LIMIT_ENABLED=false`

## Role O (Owner)
- Создать компанию -> после создания сразу видны кнопки компании (генерация кодов/назад).
- Сгенерировать инвайт-коды -> передать один код U.
- Добавить консультанта по Telegram ID -> в списке консультантов появляется `✅`.
- Удалить консультанта -> в списке остаётся как `⛔`.
- Восстановить консультанта -> снова `✅`.
- Открыть `Mental Feedback` -> выбор периода и выгрузка `feedback_*.txt`.
- Открыть счётчики -> проверить неделя/месяц/всё время и режимы total/by_consultant.
- **WHO-5 (статистика)** -> после хотя бы одного прохождения опроса пользователем: отображаются число ответов и средний сырой балл; при отсутствии данных — пустое сообщение без ошибки.

## Role U (User)
- `/start` -> выбор языка -> ввод инвайт-кода -> главное меню U.
- `Неотложная Mental Health` -> приходит дружелюбный ответ ожидания.
- `Сессия с Consultant` -> запрос -> выбор слота -> подтверждение.
- `Мои сессии` -> видна ближайшая подтверждённая сессия.
- Перенос из `Мои сессии` -> выбрать новый слот -> подтвердить перенос.
- После сессии: кнопки `Сессия была/не состоялась` работают.
- При `Сессия была` -> шкала feedback 1..10, затем комментарий (по желанию).
- **Полезные материалы** -> листание страниц (вперёд/назад), возврат в главное меню.
- **Wellbeing (WHO-5)** -> экран ввода -> старт -> 5 ответов по шкале 0–5 -> итог с суммой/индексом и текстом рекомендации; «Отмена» или «Главное меню» выходят без зависаний.

## Role C (Consultant)
- `/start` консультанта -> меню C доступно.
- Расписание: добавить/убрать слоты, сохранить.
- Срочные запросы: принять запрос, завершить диалог.
- Удалить свой аккаунт (в настройках) -> не падает, переводит в онбординг.

## Критичный сценарий удаления C
- Если у C есть будущие сессии U:
  - после удаления C будущие сессии U должны стать отменёнными;
  - новые записи к этому C не должны появляться;
  - восстановление C у O возвращает активность, но отменённые сессии не “оживают” автоматически.

## Новые сценарии — подробный чеклист (WHO-5, материалы, статистика)

*Прогоняйте под аккаунтом с нужной ролью. Удобно отмечать `[x]` и в конце написать: «всё ок» или что именно отличается от ожидания (скрин/текст сообщения).*

**Одно активное inline-сообщение:** в истории чата могут остаться старые сообщения с кнопками — клиент Telegram всё равно показывает их «живыми». **Сервер** обрабатывает только нажатия на сообщение, которое сейчас является **якорем** UI (последнее актуальное меню/опрос после `touch`); иначе показывается всплывающее предупреждение (`ui_use_latest_message` / аналог). **Исключения** (работают с любого сообщения): `menu:main`, `c_menu:main`, `o_menu:main`, `skip`, префиксы вроде `feedback:`, `urgent_accept:`, напоминаний и др. — полный список в [`bot/services/ui_surface.py`](bot/services/ui_surface.py). При **«Главное меню»** на старом экране дополнительно вызывается синхронизация текста и клавиатуры на сообщении-якоре (`sync_ui_surface_if_other` в [`bot/handlers/common.py`](bot/handlers/common.py)).

### Подготовка к этому блоку
- [ ] Бот и БД запущены, в логах бота нет traceback при старте.
- [ ] Есть тестовый **User** (прошёл онбординг, видит главное меню пользователя).
- [ ] Есть **Owner** (ваш ID в `OWNER_TELEGRAM_IDS`, видит меню владельца).
- Для проверки **статистики Owner** сначала хотя бы раз полностью пройдите опрос WHO-5 под User (сценарий B или C ниже).

### A. User — «Полезные материалы» (`📖 Полезные материалы`)

**Цель:** три страницы контента, навигация, возврат в меню без «залипания» FSM.

| Шаг | Действие | Ожидаемое поведение |
|-----|----------|---------------------|
| A1 | В главном меню User нажать **«📖 Полезные материалы»** | Сообщение **заменяется** (тот же «чат», не новое сообщение сверху). Текст начинается с заголовка про полезные материалы, есть пояснение, что это не замена терапии. Оформление — жирные фрагменты как в Markdown (`*текст*`). |
| A2 | Посмотреть кнопки внизу на **1-й странице** | Есть **«Вперёд ➡️»** и **«🏠 Главное меню»**. Кнопки **«⬅️ Назад»** на первой странице **нет** (некуда листать назад). |
| A3 | Нажать **«Вперёд ➡️»** | Текст меняется на **вторую** тему (сон). Снизу есть **«⬅️ Назад»**, **«Вперёд ➡️»**, **«🏠 Главное меню»**. |
| A4 | Ещё раз **«Вперёд ➡️»** | Текст — **третья** тема (поддержка). Есть **«⬅️ Назад»** и **«🏠 Главное меню»**; кнопки **«Вперёд»** уже **нет** (это последняя страница). |
| A5 | **«⬅️ Назад»** дважды | Возврат на 2-ю, затем на 1-ю страницу; на 1-й снова только «Вперёд» и «Главное меню». |
| A6 | С любой страницы нажать **«🏠 Главное меню»** | Сообщение меняется на **главное меню User** со всеми привычными кнопками (в т.ч. снова видны «Полезные материалы» и «Wellbeing»). **Нет** зависшего состояния: следующие сообщения в чате не воспринимаются ботом как «продолжение материалов». |

**Чекбоксы для себя:**
- [ ] A1 — вступление и формат текста ок
- [ ] A2–A4 — листание и набор кнопок на каждой странице совпадает с таблицей
- [ ] A5 — назад работает
- [ ] A6 — возврат в главное меню полный и предсказуемый

### B. User — «Wellbeing (WHO-5)» — полное прохождение

**Цель:** 5 вопросов, шкала 0–5, итог с суммой, индексом % и рекомендацией; результат сохраняется в БД (проверка косвенно через Owner, сценарий D).

| Шаг | Действие | Ожидаемое поведение |
|-----|----------|---------------------|
| B1 | **«📊 Wellbeing (WHO-5)»** в главном меню | Текст про **экспресс-опрос WHO-5**, 5 вопросов, шкала 0–5, дисклеймер «не диагноз». Кнопки: **«▶️ Начать опрос»** и **«🏠 Главное меню»**. |
| B2 | **«▶️ Начать опрос»** | Появляется **вопрос 1/5** (про бодрость и настроение). Под сообщением **шесть** кнопок с цифрами **0, 1, 2, 3, 4, 5** в один ряд и отдельной строкой **«❌ Отмена»**. **Ноль в ряду обязателен:** по WHO-5 крайние значения шкалы — *0* («никогда» / минимум проявления) и *5* («всегда» / максимум); это не ошибка вёрстки. |
| B3 | Ответить на вопросы **1–5** любой последовательностью цифр | После каждого нажатия текст **сразу** меняется на следующий вопрос (2/5 … 5/5). Шкала и «Отмена» на каждом шаге те же. |
| B4 | После ответа на **5-й** вопрос | Сообщение с **итогом:** строка «Результат WHO-5», **сумма баллов из 25**, **индекс 0–100** (формула в боте: сумма × 4). Ниже — **один из трёх** текстов рекомендации (зависит от уровня: более поддерживающий при низком индексе, нейтральнее при среднем, позитивный при высоком). |
| B5 | Кнопки под итогом | Только **«🏠 Главное меню»** (других кнопок нет). |
| B6 | **«🏠 Главное меню»** | Снова полное главное меню User. |

**Чекбоксы:**
- [ ] B1 — вступление и кнопки
- [ ] B2–B3 — все пять шагов без пропусков и дублей вопросов
- [ ] B4 — сумма и процент выглядят согласованно (например, все ответы «5» → сырой 25, индекс 100)
- [ ] B5–B6 — выход в меню

### C. User — WHO-5: отмена и выход в меню

**Цель:** прервать опрос без ошибок и без «залипшего» состояния.

| Шаг | Действие | Ожидаемое поведение |
|-----|----------|---------------------|
| C1 | Снова открыть **Wellbeing** → **«Начать опрос»** → на **любом** из вопросов 1–5 нажать **«❌ Отмена»** | Текст меняется на **главное меню User** (как после обычного «Главное меню»), полный набор кнопок. |
| C2 | На экране **вступления** WHO-5 (ещё **до** «Начать опрос») нажать **«🏠 Главное меню»** | Открывается **главное меню User** со всеми кнопками. |
| C3 | (По желанию) Снова: Начать опрос → на вопросе **ничего не отвечая**, прокрутить чат вверх и нажать **«🏠 Главное меню»** на **старом** сообщении (если оно осталось в истории) | **Ожидается:** сообщение, на котором нажали «Главное меню», перерисовывается в главное меню; сообщение с **текущим** вопросом (если оно было якорем опроса) **синхронизируется** на то же главное меню — не остаётся отдельного «живого» экрана с вопросом WHO-5. FSM сброшен. **Отдельно:** если нажать на **старом** сообщении цифру **0–5** или **«Отмена»** (не «Главное меню»), ожидается **алерт** «используйте последнее сообщение» — ответ опроса не засчитывается. |

**Чекбоксы:**
- [ ] C1 — отмена с вопроса возвращает в главное меню
- [ ] C2 — «Главное меню» с экрана вступления WHO-5 работает
- [ ] C3 — опционально: «Главное меню» со старого экрана + синхрон якоря; цифры/«Отмена» на неякорном сообщении дают алерт (см. строку C3)

### D. Owner — «WHO-5 (статистика)» (`📊 WHO-5 (статистика)`)

**Цель:** пустое состояние и состояние с данными после прохождений User.

| Шаг | Действие | Ожидаемое поведение |
|-----|----------|---------------------|
| D0 (опционально) | Свежая БД, **ни разу** не проходили WHO-5 | Пропустите или выполните: открыть пункт статистики — должен быть сценарий D1. |
| D1 | Под Owner открыть **«📊 WHO-5 (статистика)»**, если в БД **нет** записей WHO-5 | Текст вроде **«Пока нет сохранённых прохождений WHO-5»** (иконка почты/пусто в тексте). Кнопка **«🏠 Главное меню»** (меню владельца). **Нет** traceback в логах. |
| D2 | Под User **один раз полностью** пройти WHO-5 (сценарий B), затем под Owner снова **«📊 WHO-5 (статистика)»** | Текст: **число прохождений** (≥ 1) и **средний сырой балл** (0–25, один знак после запятой). Кнопка **«🏠 Главное меню»**. |
| D3 | Повторить полное прохождение WHO-5 вторым User *или* второй раз тем же User | Счётчик прохождений **увеличился**; среднее **пересчиталось** (не обязательно вручную считать — достаточно, что числа обновились логично). |

**Чекбоксы:**
- [ ] D1 — пустая статистика корректна
- [ ] D2 — после одного прохождения видны count и avg
- [ ] D3 — после второго прохождения статистика обновилась
